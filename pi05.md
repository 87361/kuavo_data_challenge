# PI05 Docker 环境依赖分析

## 一、关键发现：存在依赖缺失问题

### 1. transformers 缺失（严重问题）

**问题**：`requirements_total.txt` 中**没有 transformers**，但 PI05 必须依赖它。

PI05 使用 PaliGemma 模型，需要特殊版本的 transformers：

```python
# third_party/lerobot/src/lerobot/policies/pi05/modeling_pi05.py 第35行
from transformers.models.paligemma.modeling_paligemma import PaliGemmaForConditionalGeneration
```

**要求的版本**（来自 lerobot pyproject.toml 第121行）：

```toml
pi = ["transformers @ git+https://github.com/huggingface/transformers.git@fix/lerobot_openpi"]
```

### 2. peft 在 requirements_total.txt 中

```txt
# requirements_total.txt 第142行
peft>=0.10.0
```

这个依赖已存在，但需确认打包的 conda 环境中已安装。

### 3. Dockerfile 不会安装可选依赖

当前 Dockerfile 执行的是：

```bash
pip install -e .
cd ./third_party/lerobot && pip install -e .
```

这**不会**安装 `[pi]` 可选依赖，因此不会自动安装 transformers。

## 二、解决方案

### 方案 A：修改 requirements_total.txt（推荐）

在 `requirements_total.txt` 中添加 transformers：

```txt
# 添加到 requirements_total.txt
transformers>=4.57.0  # 或者使用特定版本
```

**或者**使用 lerobot 推荐的特殊分支：

```txt
transformers @ git+https://github.com/huggingface/transformers.git@fix/lerobot_openpi
```

### 方案 B：修改 Dockerfile

在 Dockerfile 中安装 `[pi]` 可选依赖：

```dockerfile
# 修改第54行
cd ./third_party/lerobot && pip install -e ".[pi]" && \
```

### 方案 C：确保预打包的 conda 环境包含依赖

在打包 `icra_env.tar.gz` 之前，确保环境中已安装：

```bash
# 在打包前验证
python -c "import transformers; print(transformers.__version__)"
python -c "import peft; print(peft.__version__)"
python -c "from lerobot.policies.pi05.modeling_pi05 import PI05Policy; print('PI05 OK')"
```

## 四、推荐操作步骤

1. **在远程服务器验证依赖**：
  ```bash
   conda activate your_env
   python -c "import transformers, peft; print('OK')"
  ```
2. **如果 transformers 缺失，安装它**：
  ```bash
   pip install transformers>=4.57.0
   # 或使用特殊分支
   pip install git+https://github.com/huggingface/transformers.git@fix/lerobot_openpi
  ```
3. **更新 requirements_total.txt**：
  ```bash
   pip freeze | grep transformers >> requirements_total.txt
  ```
4. **重新打包 conda 环境**：
  ```bash
   conda pack -n your_env -o icra_env.tar.gz
  ```
5. **本地同步代码后 build**：
  ```bash
   docker build -t sn4_task1 .
   docker save -o sn4_task1.tar sn4_task1:latest
  ```

## 五、PaliGemma Tokenizer 访问权限问题

### 问题描述

PI05 使用 `google/paligemma-3b-pt-224` tokenizer，这是 Google 的受限模型，需要：

1. 在 HuggingFace 上申请访问权限：[https://huggingface.co/google/paligemma-3b-pt-224](https://huggingface.co/google/paligemma-3b-pt-224)
2. 使用 `huggingface-cli login` 登录

代码位置：

```python
# third_party/lerobot/src/lerobot/policies/pi05/processor_pi05.py 第145行
TokenizerProcessorStep(
    tokenizer_name="google/paligemma-3b-pt-224",
    ...
)
```

### 当前服务器状态

远程服务器已有 tokenizer 缓存（21M）：

```
~/.cache/huggingface/hub/models--google--paligemma-3b-pt-224/
```

### 解决方案

#### 方案 1：将 tokenizer 缓存打包进 Docker（推荐）

修改 Dockerfile，将 HuggingFace 缓存复制到容器中：

```dockerfile
# 在 COPY . . 之后添加
COPY .cache/huggingface /root/.cache/huggingface
```

或者在打包 conda 环境前，将缓存复制到项目目录：

```bash
# 在服务器上执行
mkdir -p /root/autodl-tmp/kuavo_data_challenge/.cache/huggingface
cp -r ~/.cache/huggingface/hub/models--google--paligemma-3b-pt-224 \
      /root/autodl-tmp/kuavo_data_challenge/.cache/huggingface/hub/
```

#### 方案 2：离线加载 tokenizer

修改代码使用本地 tokenizer 路径，而非从 HuggingFace 下载。

#### 方案 3：在 Docker 中配置 HF_TOKEN

如果评测环境有网络访问：

```dockerfile
ENV HF_TOKEN=your_token_here
```

### 关键提醒

- **tokenizer 只有 21M**，打包进 Docker 影响不大
- **模型权重已在训练输出中**（~8.6GB），不需要再下载 PaliGemma 模型权重
- Docker 运行时如果没有 tokenizer 缓存且无法访问 HuggingFace，会报错

## 六、结论

**需要处理两个问题才能成功运行：**

1. **transformers 依赖**：确保 `icra_env.tar.gz` 中已安装 transformers
2. **PaliGemma tokenizer**：将 tokenizer 缓存打包进 Docker，或配置 HF_TOKEN

**推荐操作**：

```bash
# 1. 将 tokenizer 缓存复制到项目目录
mkdir -p .cache/huggingface/hub
cp -r ~/.cache/huggingface/hub/models--google--paligemma-3b-pt-224 .cache/huggingface/hub/

# 2. 修改 Dockerfile 添加缓存复制
# 3. 确保 requirements_total.txt 包含 transformers
# 4. 打包并测试
```

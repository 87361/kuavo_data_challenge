2. 动作执行方式
有两种执行方式：
A. 同步执行（Action Queue）
实现位置： third_party/lerobot/src/lerobot/policies/pi05/modeling_pi05.py:1174-1184
def select_action(self, batch):    if len(self._action_queue) == 0:        actions = self.predict_action_chunk(batch)[:, : self.config.n_action_steps]        self._action_queue.extend(actions.transpose(0, 1))    return self._action_queue.popleft()
特点：
队列为空时触发推理
一次推理填充 n_action_steps 个动作到队列
逐个取出执行
B. 异步执行（Receding Horizon）
实现位置： kuavo_deploy/utils/async_manager.py
关键类： AsyncInferenceManager
执行流程：
Bootstrap：首次同步推理第一个 chunk
执行当前 chunk：从 current_chunk 逐个取出动作
异步推理：在 chunk_index == n_action_steps - overlap_steps 时启动下一个 chunk 的推理
Chunk 切换：当前 chunk 执行完后切换到预计算的 next_chunk
关键方法：
should_launch_next_inference() (第124-133行)：判断是否启动下一个推理
get_action() (第204-266行)：主接口，管理 chunk 切换和异步推理

3. 控制频率配置
配置位置：
configs/deploy/kuavo_env.yaml (第22行)
kuavo_deploy/config.py (第55行)
关键参数：
ros_rate: 10  # 推理速度 (Hz)
实现细节：
环境配置中的 ros_rate 控制控制循环频率
默认 10 Hz（每步 100ms）
在 KuavoBaseRosEnv 中使用 rate.sleep() 维持频率
推理频率计算：
同步模式：推理频率 = ros_rate / n_action_steps
例如：ros_rate=10, n_action_steps=50 → 推理频率 = 0.2 Hz（每 5 秒推理一次）
异步模式：推理频率可更高（通过重叠执行隐藏延迟）

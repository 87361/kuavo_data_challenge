PI05模型推理关键流程分析报告
1. Stats加载验证问题（关键）
位置：kuavo_deploy/src/eval/sim_auto_test.py:371
preprocessor, postprocessor = make_pre_post_processors(None, Path(str(pretrained_path).split("/epoch", 1)[0]))
问题：
当 pretrained_path 不为 None 时，make_pre_post_processors 会从保存路径加载 processor（见 factory.py:209-250）
未传入 dataset_stats 或 overrides，会使用保存的 processor 中的 stats
如果保存的 stats 不正确或缺失，unnormalize 会使用错误值，导致动作异常
对比正确用法（lerobot_train.py:219-237）：
processor_kwargs["preprocessor_overrides"] = {    "normalizer_processor": {        "stats": dataset.meta.stats,  # 明确传入stats        ...    }}postprocessor_kwargs["postprocessor_overrides"] = {    "unnormalizer_processor": {        "stats": dataset.meta.stats,  # 明确传入stats        ...    }}
2. 推理流程中的unnormalize
位置：third_party/lerobot/src/lerobot/policies/pi05/modeling_pi05.py
select_action (1174-1184行)：未调用 unnormalize
predict_action_chunk (1187-1202行)：未调用 unnormalize
unnormalize 在 postprocessor 中完成：
processor_pi05.py:154-156 定义了 UnnormalizerProcessorStep
如果 postprocessor 的 stats 不正确，会导致动作值异常
注意：vlash 版本的实现不同（vlash/policies/pi05/modeling_pi05.py:1317），在 predict_action_chunk 中调用了 self.unnormalize_outputs。

## 三、功能说明

### 3.1 VLASH 优化功能

在 `configs/deploy/kuavo_env.yaml` 中配置：

```yaml
inference:
  vlash_optimization:
    # torch.compile 优化
    use_torch_compile: false          # 启用 torch.compile
    compile_mode: "max-autotune"      # 编译模式
    warmup_steps: 3                   # warmup 步数
    
    # 异步推理优化
    use_async_inference: false        # 启用异步推理
    inference_overlap_steps: 4        # 重叠步数
    n_action_steps_override: 0        # 覆盖 n_action_steps (0=使用模型配置)
    use_future_state_awareness: true  # 使用未来状态预测
```

**注意事项**：
- 异步推理需要禁用 `temporal_ensemble` 才能发挥最佳效果
- 所有优化默认禁用，保持向后兼容

## 四、性能测试结果

### Processor 开销测试

| 组件 | 耗时 | 占比 |
|-----|------|------|
| Preprocessor | 0.479 ms | 1.1% |
| Postprocessor | 0.030 ms | 0.1% |
| 纯模型推理 | 44.584 ms | 98.9% |

**结论**：Processor 开销仅占 1.1%，不是性能瓶颈。

### temporal_ensemble 消融测试

| 配置 | Jerk(平均) | Jerk(最大) | 平滑度 |
|------|-----------|-----------|--------|
| disabled (禁用) | 5.36 | 40.54 | 很差 |
| positive_0.01 | 0.69 | 15.39 | 一般 |
| **negative_0.1** | **0.32** | **6.84** | **良好** |

**结论**：当前配置 `-0.1` 是最优的，不应禁用 temporal_ensemble。

## 五、在公共电脑运行推理指南

### 环境准备

```bash
# 1. 克隆代码（HTTPS，无需登录）
git clone https://github.com/87361/kuavo_data_challenge.git
cd kuavo_data_challenge
git checkout pi05-feature

# 2. 初始化子模块
git submodule update --init --recursive
```

### 更新代码

```bash
git pull origin pi05-feature
git submodule update --recursive
```

### 运行推理

```bash

conda activate <your_env>

# 设置 PYTHONPATH
export PYTHONPATH="$(pwd):$(pwd)/third_party/lerobot/src:$PYTHONPATH"

# 运行推理测试

    --config configs/deploy/kuavo_env.yaml
```

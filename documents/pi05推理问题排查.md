# PI05 推理效果差问题排查计划

## 问题症状

- 动作"鬼畜"、快速抖动、像跳舞
- 完全没有抓取动作
- LoRA 3万步训练，1000条数据

---

## 嫌疑1：反归一化（Unnormalization）失效 - **最高优先级**

### 问题分析

在 `[sim_auto_test.py:371](kuavo_deploy/src/eval/sim_auto_test.py)` 中：

```python
preprocessor, postprocessor = make_pre_post_processors(None, Path(str(pretrained_path).split("/epoch", 1)[0]))
```

关键发现：

- 当 `key not in self._tensor_stats` 时，`[normalize_processor.py:306](third_party/lerobot/src/lerobot/processor/normalize_processor.py)` 会 **直接返回原始tensor，不做任何unnormalize**
- 这意味着模型输出的 `[-1, 1]` 范围值会被当作关节弧度值直接使用
- 结果：关节角度极小或错误 -> 机器人鬼畜抖动

### 排查步骤

**步骤1：检查保存的processor文件是否存在**

```bash
# 检查训练输出目录
ls -la outputs/train/task1/pi05/run_20260203_194626/

# 应该存在以下文件：
# - policy_preprocessor.json
# - policy_postprocessor.json  
# - *_normalizer_processor.safetensors
# - *_unnormalizer_processor.safetensors
```

**步骤2：检查stats是否正确保存**

```bash
# 查看postprocessor配置
cat outputs/train/task1/pi05/run_20260203_194626/policy_postprocessor.json

# 查看safetensors文件大小（应该不为空）
ls -lh outputs/train/task1/pi05/run_20260203_194626/*.safetensors
```

**步骤3：添加调试代码验证stats加载**

在 `[sim_auto_test.py](kuavo_deploy/src/eval/sim_auto_test.py)` 第371行后添加：

```python
# 调试：打印postprocessor的stats
for step in postprocessor._steps:
    if hasattr(step, '_tensor_stats'):
        print(f"Step: {step.__class__.__name__}")
        print(f"Stats keys: {step._tensor_stats.keys()}")
        if 'action' in step._tensor_stats:
            print(f"Action stats: {step._tensor_stats['action']}")
        else:
            print("WARNING: 'action' key not in stats!")
```

**步骤4：验证推理时的action值范围**

在推理循环中添加：

```python
# 在 policy.select_action() 后
action = postprocessor(raw_action)
print(f"Raw action range: [{raw_action.min():.4f}, {raw_action.max():.4f}]")
print(f"Unnormalized action range: [{action.min():.4f}, {action.max():.4f}]")
```

如果两者几乎相同，说明unnormalize没有生效。

## 嫌疑3：CausalLMOutputWithPast 修复

### 分析结论

修复代码位于 `[modeling_pi05.py:460-466](third_party/lerobot/src/lerobot/policies/pi05/modeling_pi05.py)` 是 **正确的**：

```python
if hasattr(suffix_output, 'last_hidden_state'):
    suffix_output = suffix_output.last_hidden_state
elif hasattr(suffix_output, 'hidden_states') and suffix_output.hidden_states:
    suffix_output = suffix_output.hidden_states[-1]
```

`hidden_states[-1]` 与 `last_hidden_state` 在Gemma模型中等价，不会导致问题。

## 快速修复方案（如果确认是Stats问题）

在 `[sim_auto_test.py](kuavo_deploy/src/eval/sim_auto_test.py)` 中强制加载数据集stats：

```python
from lerobot.datasets import LeRobotDataset, LeRobotDatasetMetadata
from lerobot.processor.normalize_processor import hotswap_stats

# 加载数据集的stats
dataset_meta = LeRobotDatasetMetadata(
    repo_id="your_dataset_repo",
    root=Path("path/to/dataset")
)

# 替换postprocessor中的stats
postprocessor = hotswap_stats(postprocessor, dataset_meta.stats)
```
